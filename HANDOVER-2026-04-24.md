## What We Did Today (Full List)

### Fix 1 — Patreon OAuth Data Loss

**The bug:** User generates a listing → clicks "Sign in with Patreon" → gets redirected to Patreon → comes back → listing is gone AND they're still not signed in.

**Root cause:** Both sign-in buttons were plain `<a href>` links causing full-page navigation. React re-mounts on return, all in-memory state (results, imagePreview) lost. Only auth state (isPro, patreonTier) was persisted to localStorage.

**The fix:**
1. Both "Sign in with Patreon" buttons now have an `onClick` that saves `results` and `imagePreview` to `sessionStorage` (key: `spicylister_pending_results`) before navigating.
2. The OAuth callback `useEffect` (handles `?code=` on return) now also restores from sessionStorage after setting auth state, then clears it.

**Files changed:** `src/App.js`
- Lines ~1176 and ~1244: onClick handlers on both sign-in `<a>` tags
- Lines ~457-465: sessionStorage restore in OAuth callback useEffect

---

### Fix 2 — eBay Tab-Switch Data Loss

**The bug:** User generates listing → clicks eBay button (opens new tab with `window.open(_blank)`) → browses eBay → comes back to SpicyLister tab → listing is gone.

**Root cause:** Mobile browsers kill background tabs to save memory. When the user returns, React re-mounts from scratch and all state is lost.

**The fix:**
1. New `useEffect` that auto-saves `results` + `imagePreview` to `localStorage` (key: `spicylister_auto_save`) whenever they change, with a 4-hour TTL timestamp. If quota is exceeded by the image, retries saving just the results without imagePreview.
2. New `useEffect` on mount that checks `localStorage` for saved data under 4 hours old and restores it.
3. `resetApp()` now calls `localStorage.removeItem('spicylister_auto_save')` so clicking "New" gives a genuinely fresh start.

**Files changed:** `src/App.js`
- Lines ~400-436: two new useEffects (restore on mount, save on change)
- Line ~622: localStorage.removeItem in resetApp

**Note:** This also makes the Patreon flow more robust — results survive the redirect via localStorage as a second layer even if sessionStorage doesn't fire.

### Feature 1 — OID Network Recommendation

**What it does:** After a listing is generated, the app scores the title, category, and description against 15 oids using keyword matching. If an item scores ≥ 2 points, a tappable indigo banner appears in the results linking to the matching oid.

**Scoring:** Title keyword hits = 3 points, body/category hits = 1 point. Threshold = 2.

**Where it appears:** In the results section, between the packaging card and the FeelFamous button.

**The constant:** `OID_NETWORK` at `src/App.js` line ~241. Array of 15 oids, each with: name, emoji, url, tagline, keywords[].

**The function:** `getOidRecommendation(results)` at `src/App.js` line ~258. Returns matching oid object or null.
